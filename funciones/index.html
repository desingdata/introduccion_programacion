1. Declaración de Funciones

Cómo definir una función por primera vez

Declaración clásica function declaration

Usa la palabra clave function. Puede llamarse antes de declararse (hoisting).

// "function" inicia la definición; "saludar" es el nombre
function saludar() {
  // El cuerpo va entre llaves {}
  console.log("¡Hola, mundo!");
}
// Invocar la función añadiendo "()"
saludar();  // → ¡Hola, mundo!

Expresión de función function expression

La función se asigna como valor a una variable. No tiene hoisting.

// La función anónima se guarda en la variable
const despedir = function() {
  return "¡Hasta luego!";
};
console.log(despedir());  // → ¡Hasta luego!
🔧

2. Parámetros y Argumentos

Enviar datos hacia dentro de la función

Parámetros simples parameters

Actúan como variables locales que reciben valores al momento de la llamada.

// "nombre" y "edad" son parámetros (variables locales)
function presentar(nombre, edad) {
  // Template literal: interpolación con ${}
  return `Soy ${nombre}, tengo ${edad} años`;
}
// "Ana" y 22 son los argumentos (valores reales)
console.log(presentar("Ana", 22));
// → Soy Ana, tengo 22 años

Parámetros por defecto default params

Si el argumento no se pasa, se usa el valor predefinido en la firma.

// "mundo" es el valor por defecto de "nombre"
function hola(nombre = "mundo") {
  return `¡Hola, ${nombre}!`;
}
hola();          // → ¡Hola, mundo!
hola("Carlos"); // → ¡Hola, Carlos!

3. Valor de Retorno

La función produce un resultado usable

return simple return

Devuelve un único valor. Nada después de return se ejecuta.

// Calcula el área de un rectángulo
function area(base, altura) {
  return base * altura; // ← sale aquí
}
const r = area(5, 3);
console.log(r); // → 15

4. Tipos de Funciones

Formas alternativas de escribir funciones en JS

Arrow Function ES6 =>

Sintaxis compacta. Sin cuerpo retorna implícitamente. No tiene this propio.

// Arrow: "=>" reemplaza la palabra "function"
const doble = (n) => n * 2; // return implícito
// Con cuerpo multilínea: necesita {} y return
const potencia = (base, exp) => {
  return base ** exp;
};
doble(4);        // → 8
potencia(2,8); // → 256

Callback función como argumento

Una función que se pasa como argumento a otra para ejecutarse después.

// "operacion" es un parámetro que espera una función
function operar(a, b, operacion) {
  return operacion(a, b); // invoca el callback
}
operar(6, 2, (x,y) => x + y); // → 8
operar(6, 2, (x,y) => x * y); // → 12

IIFE invocación inmediata

Se declara y ejecuta al mismo tiempo. Crea un ámbito privado aislado.

// () externos invocan la función al instante
const resultado = (function() {
  const secreto = 42; // variable privada
  return secreto * 2;
})();
console.log(resultado); // → 84

Recursiva recursión

La función se llama a sí misma. Necesita un caso base para no buclar infinitamente.

// Factorial: n! = n × (n-1) × … × 1
function factorial(n) {
  // Caso base: detiene la cadena de llamadas
  if (n <= 1) return 1;
  // Caso recursivo: n × factorial(n-1)
  return n * factorial(n - 1);
}
factorial(5); // → 120

5. Ámbito (Scope) y Closure

Dónde viven y cuánto duran las variables

Global vs Local scope

Variables globales: visibles en todo el programa. Locales: solo dentro de su función.

const global = "Soy global"; // accesible en todo

function ejemplo() {
  const local = "Soy local"; // solo aquí dentro
  console.log(global); // ✓
  console.log(local);  // ✓
}
ejemplo();
// console.log(local) → ❌ ReferenceError

Closure — función que recuerda closure

Función interna que conserva acceso a variables del ámbito padre, incluso después de que este termine.

// Fábrica de contadores independientes
function crearContador() {
  let cuenta = 0; // variable "capturada"
  return () => {
    cuenta++;          // recuerda "cuenta"
    return cuenta;
  };
}
const contar = crearContador();
contar(); // → 1
contar(); // → 2
